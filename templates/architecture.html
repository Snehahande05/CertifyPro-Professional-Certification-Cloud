AWS Integration Detail
Amazon EC2

Hosts the Python Flask application inside a Docker container. Managed with auto-scaling to maintain performance.

Amazon RDS (MySQL)

Relational Database Service hosting MySQL. Stores structured schemas for users, candidates, and workflows with automated daily backups.

Amazon S3

Simple Storage Service used to hold uploaded documents. Supports pre-signed URLs to enforce download authorization limits.

Amazon CloudWatch

Monitors application CPU/RAM metrics and gathers syslogs dynamically to trigger warnings if resources breach threshold states.

VPC Networks & Security Groups
Security Group Inbound Rules (Ingress) Outbound Rules (Egress)
ALB-SG (Load Balancer) HTTP (80) & HTTPS (443) from 0.0.0.0/0 Forward to EC2-SG on Port 5001
EC2-SG (Flask App Server) Port 5001 from ALB-SG; SSH (22) from Admin IP All Traffic to Internet (S3, CloudWatch API)
RDS-SG (MySQL Database) MySQL Port (3306) restricted to EC2-SG only None (DB remains fully isolated in Private Subnet)

Public subnets route outbound traffic via the Internet Gateway (IGW). Private subnets communicate through NAT Gateways for system updates.

RDS MySQL SSL/TLS Connectivity

Connecting to AWS RDS MySQL from our EC2 Flask application utilizes SSL verification to encrypt data in transit. Below is the connection parameter setup required in production: 

# Production secure db.py connector configuration
ssl_config = {
    'ssl_ca': '/app/certs/global-bundle.pem',
    'ssl_verify_cert': True
}
conn = mysql.connector.connect(
    host=DB_HOST,
    user=DB_USER,
    password=DB_PASSWORD,
    database=DB_NAME,
    ssl_config=ssl_config
)
Download the CA Bundle: wget https://truststore.pki.rds.amazonaws.com/global/global-bundle.pem
IAM Role & Permission Policies (EC2 Instance Profile)

Rather than hardcoding credentials, the EC2 instance profile should be assigned an IAM Role with the following custom policy to allow uploads to S3 and streaming to CloudWatch: 

{
    "Version": "2012-10-17",
    "Statement": [
        {
            "Sid": "S3BucketAccess",
            "Effect": "Allow",
            "Action": [
                "s3:PutObject",
                "s3:GetObject",
                "s3:DeleteObject"
            ],
            "Resource": "arn:aws:s3:::certifypro-document-vault/*"
        },
        {
            "Sid": "CloudWatchLoggingAndMetrics",
            "Effect": "Allow",
            "Action": [
                "logs:CreateLogGroup",
                "logs:CreateLogStream",
                "logs:PutLogEvents",
                "cloudwatch:PutMetricData"
            ],
            "Resource": "*"
        }
    ]
}
